Each component below is a standalone unit — geometry, behaviour, and dependencies described in isolation, so you can lift any single entry into another stylesheet without inheriting a name that ties it to where Cicrus happens to use it. Names describe what the thing is; descriptions cover its shape, its variants, and the smallest sensible context for which it's the right answer.

01Layout Foundation 04/ENT
01·01 Canvaspage wrapper · 1280px cap

Centered content well that caps at 1280px, breathes 32px on every side, and fades a four-pixel rise on mount. The outermost wrapper around any page-level layout. A --wide 1600px variant and a --full uncapped variant loosen the constraint for graph canvases and dense data views.

01·02 Page Headerbaseline-aligned title row

Flex row that baselines a display-font title against a mono caps meta string and wraps to two lines below 700px. The visual top of any page; never nest one inside a panel — it is by definition the highest element on the screen.

01·03 Sectioneyebrow + slot · 48px margin

Vertical-rhythm primitive. A mono caps eyebrow above a content slot, with 16px beneath the label and 48px beneath the slot. The simplest way to break a long page into legible bands without inventing new dividers.

01·04 Action Barflex row · wraps

Horizontal flex row that sits above a data view and holds filter triggers, search fields, and primary actions. A --spread variant splits its children into left and right clusters at opposing edges; otherwise everything flows left and wraps cleanly on narrow viewports.

02Surfaces Foundation 03/ENT
02·01 Panel1px border · 6px radius · 16px pad

The default container for grouped content. Hairline border on the page surface, six-pixel radius, sixteen-pixel padding. A clickable variant promotes the border on hover with no shadow or scale — the only feedback is a single-stop colour shift.

02·02 Inset Panelraised-surface fill

Same chassis as Panel with a lighter fill, suited for content that nests inside another Panel — drawer interiors, dropdown menus, items inside a Column Stack. The contrast step keeps the inner content readable without inventing a second border weight.

02·03 Divider1px hairline · 24px margin

A one-pixel hairline in the border token with 24px of vertical clearance. The only horizontal rule in the system. Used to separate sibling sections inside a single panel; never decoratively.

03Typography Foundation 04/ENT
03·01 Display NumeralDoto · 36–72px

Dot-matrix face. Reserved for hero numbers, clocks, the wordmark, and figure-tile values. Loses legibility below 24px — never use it for paragraphs, labels, or anything you expect a user to read more than a single token of.

03·02 HeadingSpace Grotesk · 24 / 18px

The everyday header pair. Weight 500 at 24px for top-of-component titles; weight 400 at 18px for subheadings beneath. Tracking is tightened on both to recover the mass that Grotesk's neutral letterforms naturally shed.

03·03 ParagraphSpace Grotesk · 16 / 14px

The default for running text, 16px at 1.5 line-height. A 14px compact variant earns its place inside dense rows — table cells, log rows, status tiles — where the larger size would feel loud.

03·04 EyebrowSpace Mono · 10–11px · 0.08em · UPPER

The voice of metadata. Mono, ALL CAPS, eleven-pixel with 0.08em tracking; a ten-pixel variant for inline use. If a string repeats across screens and isn't editorial — timestamp, count, status, key — it almost always wants this.

04Actions Interactive 05/ENT
04·01 Solid Actionfilled pill · 40px

Pill button with the display colour as its fill and black for the label. The single highest-weight action on any screen — one per view, only when a commitment is being asked of the user. Hover steps the fill down to primary text.

04·02 Outlined Actiontransparent pill · 1px border

Transparent pill with a visible border. The supporting action beside a Solid Action, and the default chassis for filter rows where every button has equal weight. Hover promotes the border to primary text.

04·03 Quiet Actionborderless · secondary text

No border, no fill, secondary-text label. The lightest possible affordance — table-row actions, dismiss links, inline navigation inside dense layouts. Reads as a link until you hover it; only colour responds.

04·04 Destructive Actionoutlined pill · accent-red

Pill outlined in the accent-red token. Reserved for irreversible actions — delete, force-kill, abandon. A subtle accent-tint fill appears only on hover, so the button reads as warning at rest and as commitment under the cursor.

04·05 Dropdown Triggercompact pill · chevron

Smaller pill — 32px tall, mono caps, chevron at the trailing edge. Sits in an Action Bar to expose filter, sort, or grouping menus. Reads as a control, not a commitment; never use one to submit a form.

05Form Controls Interactive 04/ENT
05·01 Line Fieldunderline · no box

Single-line input with one hairline border below the text and no other chrome. Placeholder renders mono caps. On focus the underline promotes to primary text; on error it tints accent-red, with an optional bracketed mono error string beneath.

05·02 Capsule Fieldsurfaced pill · prefix slot

Full-pill input on the page surface, hairline-bordered. Carries an optional mono prefix slot — e.g. FIND, FILTER, : — that telegraphs what the field accepts. Use for prominent search bars and command rows; not for forms.

05·03 Toggle36×20 track · 14px thumb

Pill-shaped track with a circular thumb. Off: thumb in disabled text, track on the visible border. On: thumb and track both promote to primary text. Same visual language as the global light/dark mode switch — never invent a second toggle style.

05·04 Segmented Controlpill of 2–4 buttons

Pill split into two to four mono caps segments. Active segment fills with the display colour and a black label; inactive segments are transparent and only lift to primary text on hover. Use where one option is always selected.

06Tags Taxonomy 03/ENT
06·01 Tag Pillmono caps · 10px · 1px border

The base from which every tag variant below is spun. Mono caps, 10px, 0.06em tracking, 3×10px padding, 999px radius, one-pixel hairline border. Fill is always transparent; the role token colours label and border together.

06·02 Category Tagdomain-tinted · 7 roles

Tag Pill tinted by domain: process (primary), resource (interactive blue), monitor (warning amber), steps (success green), system (disabled), recurring (faint), generic type (secondary). Labels what an object is. Cap at three per card.

06·03 Status Taglifecycle-tinted · 3 states

Tag Pill tinted by lifecycle: success (green), warning (amber), accent (red). Labels where in its lifecycle an object sits. Reserve accent for the single red item per screen — overuse drains it of meaning.

07Signals Feedback 05/ENT
07·01 Signal Dot8px circle · 4 states

Eight-pixel circle with four states: up, down (two-second pulse), warning, idle. The smallest possible status read; pair with a name for context.

07·02 Notice Barfull-width tinted strip

Full-width bar with a tinted background and matching border, a single SVG glyph at the leading edge, and a single line of mono caps copy. Three tones — accent (error), warning, info — share one chassis; the colour does the talking.

07·03 Trace Blocktitle + mono red detail

A 14px body title stacked above a mono accent-red detail string with word-break: break-all, so a console paste or stack trace never blows out the layout. Designed to stack vertically — the unit of any failure log.

07·04 Animated Glyphcanvas · 96-pixel grid

Canvas-rendered dot-matrix glyph with three states — idle (wobbling putty blob), thinking (siri-style swirl with heartbeat ring), error (servo-failure shatter, warning red). Sized 32–288px. One per screen — the "moment of surprise."

07·05 Segment Bardiscrete segments · square ends

Horizontal track of equal-width segments, four to eight per row, two- to sixteen-pixel tall. Filled segments swap to display colour; status variants tint to success, warning, or danger. Use where exact pixel proportions would be over-precise — ratios, not measurements.

08Data Display Composite 04/ENT
08·01 Figure Tiledisplay value + eyebrow label

Panel stacking a Display Numeral value (36px) over an Eyebrow label, with an optional Segment Bar at the foot for ratio context. The value tints to success, warning, or danger by variant. Sits in a four- or six-column grid.

08·02 Status Tilesignal dot + name + detail

Panel pairing a Signal Dot, a 14px service name, and a mono detail block with two to four lines of context (version, host, last-check). The unit of any status grid — four to a row on desktop, one on mobile.

08·03 Metadata List160px term · mono value

Two-column <dl>. Term column is 160px wide in mono caps eyebrow style; value column is mono in primary text. The right shape for environment / version / owner blocks where alignment matters more than density.

08·04 Log Rowtime · tags · content · 3-col grid

Three-column grid — 160px timestamp, auto-width tag cluster, fluid content stack. The entire row tints on hover and bleeds 16px into the gutter so the affordance reads at a glance. The unit of any chronological feed.

09Collections Composite 02/ENT
09·01 Column Stacktitled · staggered fade-in

Vertical column of stacked cards with a mono caps title and muted count above. Auto-fades in on mount with a staggered delay across siblings. The unit of any kanban-style layout; four columns on desktop, two on tablet, one on mobile.

09·02 Index Columntitle + sublabel + slug list

A bordered column-title, an optional mono caps sublabel (e.g. 139 LISTING), and a list of mono text links separated by hairlines. Links promote to display colour on hover. The unit of any multi-column index, directory, or wiki landing.

10State & Texture Atomic 04/ENT
10·01 Empty Statecentered · 96px pad

Centered heading-light line over a mono caps tagline, padded 96px top and bottom. The shape every list collapses to when there's nothing to show — and the shape worth designing for early, since you'll see it more often than you expect.

10·02 Loader Markermono caps · ::before

Mono caps placeholder that prints [ LOADING... ] via ::before. Scaffolding, not UI — replace with content as soon as data lands. A sibling Placeholder variant prints [NOT YET IMPLEMENTED] for routes wired up but not yet built.

10·03 Dot Fieldradial-gradient · 12–20px grid

Radial-gradient background built from a single token, 0.5–2px dot at 12–20px cadence. For hero backings, empty states, and data-viz floors. The residue of the dot-matrix typography, scaled down — if everything has dots, nothing does.

10·04 WordmarkCICRUS · Doto display

The brand wordmark itself, set in Doto. Used once per surface — masthead, nav, splash. Sized 28–72px depending on context; never combined with secondary marks, never coloured outside the display token.
