Surfaces & Layout

View shell
The page wrapper. Caps width at 1280px, centres, and fades content in on mount. Use .c-view at the root of any LiveView; the --wide and --full modifiers loosen the cap when a kanban or graph needs the room.
View header
Baseline-aligned hero row that pairs a display-font title with a mono meta string. Wraps gracefully on narrow viewports. Best for top-of-page identity — never use it inside a card.
Card
The default surface for grouped content. One-pixel border on the page surface, six-pixel radius, sixteen-pixel padding. --raised swaps to the raised surface token for nested or modal contexts; --clickable tightens the hover border.
Section
Vertical rhythm primitive. A section title (mono caps, eleven-pixel) over content, with forty-eight pixels of breathing room beneath. The simplest way to keep a long page legible without inventing new dividers.
Toolbar
Horizontal flex row for filter buttons, search, and primary actions above a data view. The --spread modifier pushes .c-toolbar-left and .c-toolbar-right to opposing edges.
Dot-grid motif
A radial-gradient background built on the border token, sixteen-pixel cadence. Use sparingly: empty states, hero panels, schedule canvases. .dot-grid-strong bumps the dot to a visible pixel for emphasis surfaces.

Typography

Display
Doto, dot-matrix. Reserved for hero numbers, the clock, the home page wordmark, and stat values. Never use display type for paragraphs — the matrix legibility falls apart below twenty-four pixels.
Heading & Subheading
Space Grotesk at twenty-four and eighteen pixels. The everyday header pair for cards, modals, and section heroes. Weight is five hundred on heading, four hundred on subheading; tightened tracking on both.
Body & Body-sm
Space Grotesk at sixteen and fourteen pixels with 1.5 line-height. The default running-text style. Body-sm earns its place inside dense surfaces — table rows, card descriptions, drawer panels — where sixteen would feel loud.
Label & Label-xs
Space Mono, ALL CAPS, eleven and ten pixels with eight-percent tracking. The voice of metadata — timestamps, counts, status, kbd hints. If a string repeats across screens and isn't editorial, it probably wants this.
Section title
The mono caps eyebrow above any section. Same metrics as label but reserved positionally: one per section, sixteen pixels above the content, never beside another label.

Buttons

Primary
Solid display-coloured pill with a black label. The single highest-weight action on any screen — only one per view, and only when a commitment is being asked for.
Secondary
Transparent pill with the visible-border token and primary text. Used for the supporting action beside a primary, and as the default for top-of-page filter rows.
Ghost
No border, no background, secondary text. The lightest possible affordance — table-row actions, dismiss links, inline navigation. Pair it with hover-only colour shifts.
Danger
Outlined accent-red pill. Destructive actions only: delete, force-kill, abandon. The subtle accent tint on hover keeps it from shouting unless engaged.
Filter button
Mono caps pill with an optional chevron. Sits in toolbars to expose filter, sort, and grouping dropdowns. Smaller than a true button; reads as a control, not a commitment.

Inputs

Underline input
The default. A single hairline border below transparent text, with a mono caps placeholder. Drops into forms, modals, and inline edits without inventing chrome.
Pill input
Full-pill surfaced input for prominent search bars and command rows. Carries an optional mono prefix slot — e.g. FIND or : — to telegraph what the field accepts.

Badges

Type badges
Process, resource, monitor, steps, system, recurring. The taxonomy of objects in the workspace — used on cards, in timeline rows, in headers. Mono caps, ten-pixel, hairline border tinted by role.
Status badges
Success, warning, accent. The same pill chassis but driven by lifecycle: ok, pending, failed. Used on lanes, alerts, and inline beside a process name.
Generic badge
The unstyled base. Reach for this when adding a new taxonomy that doesn't match an existing role — and consider promoting it to a token if it appears twice.

Status & Signals

Status dot
Eight-pixel circle with four variants: up, down (pulses), warning, idle. Pair with a service name for the smallest possible status read.
Alert banner
Full-width tinted bar above a view to surface a failed dependency, a degraded surface, or an informational note. Accent, warning, and info variants share one chassis — the colour does the talking.
Failure item
The row used to log an individual failure: bold title in body type, mono accent-red detail string. Wraps long stack traces with word-break: break-all so a console paste never blows out the layout.
State glyphs
The dot-matrix iconography — built from the same Doto vocabulary as the wordmark. Used at small sizes to mark loading, success, fail, and pending states where a dot alone is too quiet.

Data Display

Stat grid & Stat
Six- or four-column grid of stat cards. Each card stacks a display-font value over a mono caps label, with an optional four-pixel segmented bar at the foot for ratios. Value variants tint for danger, success, warning.
Service card
The unit of the /status view. Status dot + name + detail block in a single card. Fits four to a row on desktop, collapses to one on mobile.
Key-value list
A two-column <dl> with a 160-pixel mono caps term and a body-mono value. The right shape for metadata blocks — environment, version, started-at, owner — where alignment matters more than density.
Timeline row
Three-column grid: time, badges, content. Whole row tints on hover and bleeds into the gutter so the affordance reads at a glance. The native unit of /timeline.

Kanban & Knowledge

Kanban column
Four-column grid (two on tablet, one on mobile) of stacked task cards. Each column header carries a mono caps title and a muted count, with the column itself fading in on mount on a staggered delay.
Knowledge columns
Three-column reading layout for the wiki. Each column has a bordered title, optional sub-label, and a list of .c-knowledge-item entries that promote to display-coloured text on hover.

Empty, Loading & Utilities

Empty state
Center-aligned heading-light line over a mono caps tagline, padded by ninety-six pixels top and bottom. The shape every list collapses to when there's nothing to show.
Loading & Not-implemented
Two mono caps placeholders that print [ LOADING... ] and [NOT YET IMPLEMENTED] via ::before. Never ship them unannotated — they're scaffolding, not UI.
Divider
A one-pixel border-token hairline with twenty-four pixels of margin above and below. The only horizontal rule you should need.
Type utilities
.c-mono, .c-display, .c-caps, plus the tone helpers .c-muted, .c-secondary, .c-primary, .c-danger, .c-success, .c-warning. Atomic, single-axis, never composed into new components.
